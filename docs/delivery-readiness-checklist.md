# Delivery Readiness Checklist

Use this checklist in the last week before a milestone/demo/release to confirm readiness.

- [ ] Acceptance criteria met for committed stories
- [ ] QA sign-off on acceptance criteria (manual or automated)
- [ ] Documentation (README, runbooks, relevant docs/) updated
- [ ] UX validation completed (if applicable)
- [ ] Performance and security scans completed and issues triaged
- [ ] Release/Deployment owner identified (Release Manager)
- [ ] Deployment window scheduled and communicated
- [ ] Dependencies to other teams confirmed and on-track
- [ ] Rollback and mitigation steps documented
- [ ] Stakeholder demo and materials scheduled

Notes:
- Assign owners for any outstanding risks and add to the Risk Register.
